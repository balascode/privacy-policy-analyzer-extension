# Contributing

Thank you for contributing.

## Scope

This repository contains extension frontend code and documentation.
Backend services can be maintained in a separate repository.

## Development Workflow

1. Create a feature branch.
2. Make focused changes.
3. Test extension behavior in browser.
4. Update docs when behavior changes.
5. Open pull request with clear summary.

## Code Standards

- Keep JavaScript logic defensive against missing fields.
- Avoid breaking API schema assumptions without doc updates.
- Keep user messages clear and actionable.

## Testing Checklist

- Context menu appears on text selection.
- Popup loads and transitions states correctly.
- Success responses render summary and clauses.
- Error responses show clear messages.
- Copy and save actions work.

## Pull Request Guidelines

Include:
- what changed
- why it changed
- how you tested
- any related issue links
